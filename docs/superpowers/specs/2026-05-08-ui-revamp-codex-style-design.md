# UI Revamp — Codex-Terminal Style

**Date:** 2026-05-08
**Status:** Draft (awaiting review)
**Scope:** `apps/web/` (web client only). Bridge protocol, stores, services, and `packages/bridge/` are not modified.

## Goal

Replace the current desktop-first, multi-pane UI in `apps/web/` with the design language of the `codex-terminal/` mockup: mobile-first single-pane layout with adaptive desktop side-rail, dark+light themes, Tailwind v4, lucide icons, motion transitions. Keep all existing features and functional wiring.

## Decisions

| Topic | Decision |
|---|---|
| Layout | Mobile-first restructure (replaces sidebar+chat+drawer) |
| Styling tech | Tailwind v4 (full adoption) |
| Feature scope | Keep all existing features; map to new nav |
| Desktop layout | Adaptive — bottom nav on mobile, left rail on `≥md`; chat optionally shows a right-side 2nd pane (FileExplorer) on desktop |
| Icons / animation | `lucide-react` + `motion` |
| Recent Files sheet | Skipped for v1 — restyle current `FileExplorer` instead |
| Settings page | Full — connection, appearance, default agent, default workspaces, profiles, accounts |
| Theme | Dark default + light mode; system fallback; persisted to localStorage |
| Projects tab | Saved/recent project paths list with reorder/remove; tap → ProjectPicker |
| Home vs Sessions | Home = launcher (`+ New Session` + history); Sessions = alive sessions only |

## Out of Scope

- Bridge protocol changes
- Store/service-layer rewrites
- New features beyond restyle (no Recent Files sheet, no extra agents, no auth changes)
- React 18 → 19 upgrade
- Light-theme palette tuning beyond initial mapping

## Architecture

### Shell

```
apps/web/src/
├── shell/
│   ├── AppShell.tsx        ← layout: NavRail + ContentPane (+ optional 2nd pane on desktop)
│   ├── NavRail.tsx         ← bottom-nav (mobile) / left-rail (≥md); 4 tabs
│   ├── ThemeProvider.tsx   ← applies data-theme to <html>; localStorage + system pref
│   ├── ViewTransition.tsx  ← motion AnimatePresence; reduced-motion aware
│   ├── BottomSheet.tsx     ← reusable mobile sheet
│   └── Modal.tsx           ← reusable centered modal
├── pages/
│   ├── Home.tsx
│   ├── Sessions.tsx
│   ├── Projects.tsx
│   ├── Settings.tsx
│   └── Session.tsx         ← chat view; replaces existing pages/Session.tsx
├── features/               ← unchanged structure; per-feature .css files removed in favor of utility classes
└── App.tsx                 ← Router; ThemeProvider; AppShell wraps nav routes
```

### Routing

```
/             → Home
/sessions     → Sessions      (NEW)
/projects     → Projects      (NEW)
/settings     → Settings      (NEW)
/session/:id  → Session       (existing path, restructured page)
*             → Navigate to /
```

`AppShell` renders `NavRail` + `<Outlet />` wrapped in `ViewTransition`. WebSocket bootstrap effect (currently in `App.tsx`) moves into `AppShell` mount effect with identical logic.

### Responsive

Breakpoint: `md` (768px).

- `< md`: bottom nav, single pane content fills viewport (`100dvh`). On `/session/:id`, NavRail hides — full-screen chat. Mobile menu (`Menu` icon in chat header) opens a `BottomSheet` containing sessions list.
- `≥ md`: left rail (~64px wide, icon-only). Single content pane fills the rest. On `/session/:id`, an optional right-side pane (FileExplorer) slides in via motion when toggled.

## Design Tokens

Tailwind v4 `@theme` block in `apps/web/src/index.css`. Common tokens shared; theme-scoped tokens via `[data-theme="dark"]` / `[data-theme="light"]`.

```css
/* Common */
--font-sans: "Inter", ui-sans-serif, system-ui, sans-serif;
--font-mono: "JetBrains Mono", ui-monospace, SFMono-Regular, monospace;
--color-accent:  #339af0;
--color-success: #40c057;
--color-warn:    #fcc419;
--color-danger:  #fa5252;

/* Dark */
[data-theme="dark"] {
  --color-bg:          #111214;
  --color-surface:     #1a1b1e;
  --color-surface-2:   #25262b;
  --color-border:      #373a40;
  --color-text:        #e5e7eb;
  --color-text-mute:   #9ca3af;
  --color-text-dim:    #6b7280;
  --color-bubble-user: #4b4461;
  --color-bubble-ai:   #2c3340;
  --color-tool-shell:  rgba(64,192,87,0.18);
  --color-tool-result: rgba(252,196,25,0.18);
}

/* Light */
[data-theme="light"] {
  --color-bg:          #fafafa;
  --color-surface:     #ffffff;
  --color-surface-2:   #f1f3f5;
  --color-border:      #dee2e6;
  --color-text:        #1a1b1e;
  --color-text-mute:   #495057;
  --color-text-dim:    #868e96;
  --color-bubble-user: #dbe4ff;
  --color-bubble-ai:   #f1f3f5;
  --color-tool-shell:  rgba(64,192,87,0.14);
  --color-tool-result: rgba(252,196,25,0.18);
}
```

Tailwind utilities: `bg-bg`, `bg-surface`, `text-text`, `border-border`, `text-accent`, etc. (generated by `@theme`).

**Typography:** Inter for body, JetBrains Mono for code/paths. Body 14–15px on mobile, 14px on desktop. Section labels: `text-xs font-bold uppercase tracking-wider text-text-dim`.

**Spacing/radius:** Tailwind defaults. Mockup uses `rounded-lg`, `rounded-xl`, `rounded-2xl`. Tap targets ≥44px.

**Motion:** `duration: 0.2`, `ease: easeOut`. View change = fade + slide-x 20px. Modal = scale 0.9→1 + fade. Sheet = translate-y. Honor `prefers-reduced-motion`.

**Theme flicker prevention:** inline `<script>` in `index.html` reads `localStorage.theme` and sets `data-theme` on `<html>` before React mounts.

## Pages

### Home (`/`)

- Big `+ New Session` button (full-width, `bg-accent`, `py-4`, `rounded-2xl`, shadow). Opens ProjectPicker modal.
- "ACTIVE SESSIONS" section: empty-state card if none; otherwise top 3 with "See all →" linking to `/sessions`.
- "HISTORY" section: closed sessions (filtered `!alive`) + archived from `useHistoryStore`. Card list with divided rows, status dot, last-seen.
- Connection / error banner pinned top by `AppShell`.

### Sessions (`/sessions`)

- Reuses `SessionList` feature, restyled, filtered to `alive=true`.
- `+ New Session` button at top.
- Empty state: "No active sessions. Start one from Home or Projects."

### Projects (`/projects`)

- New `useProjectsStore` (zustand, localStorage-persisted): list of project paths derived from session history + manual additions. Methods: `add`, `remove`, `move`, `list`. Auto-populates when new sessions open.
- Each row: project name (basename), full path (mono, truncated), session count, last-used.
- Tap → opens ProjectPicker modal pre-filled with that path.
- Reorder via lucide `ArrowUp`/`ArrowDown`; delete via `Trash2`. Long-press (mobile) reveals these inline; desktop hovers reveal them.
- `+ Add Project` button at top.

### Settings (`/settings`)

Vertically scrollable sections.

1. **Connection** — bridge URL display, status indicator, reconnect button.
2. **Appearance** — theme selector: System / Light / Dark.
3. **Default agent** — Claude / Codex segmented toggle.
4. **Default workspaces** — list of paths used to seed new sessions (existing `default-workspaces` logic). Add/edit/reorder.
5. **Profiles** — full profile manager (existing `useProfileStore`); create / edit / delete / set default.
6. **Accounts** — display from `useAccountsStore`.

### Session (`/session/:id`)

- **Header:** lucide `Menu` (mobile only — opens sessions sheet), project path (mono, truncated), session name + rename pencil, lucide `Folder` (toggles file-explorer pane).
- **Banners:** transcript-only / error (existing).
- **Chat scroll:** `MessageBubble` restyled per mockup:
  - User: `bg-bubble-user`, right-aligned, max 85% width.
  - AI: `bg-bubble-ai`, left-aligned.
  - System: centered, italic, `text-text-dim`.
  - Tool blocks: pill with lucide `Play` icon, green for `tool: shell`, gold for `tool: result`.
- **Input bar:** rounded-xl `bg-surface-2` wrapper. Textarea + bottom row: `Paperclip` (image attach), `History` (prompt-history dropdown), `Stop`, `Send`. Send is `bg-accent` when input non-empty. ResumePrompt + autocompletes anchored above input.
- **Mobile:** NavRail hidden when active. `Menu` opens sessions `BottomSheet`.
- **Desktop:** NavRail stays. FileExplorer slides in as right-side 2nd pane (no overlay).

## Error Handling, Edge Cases, Accessibility

- **Connection errors:** existing `useConnectionStore` flow. Banner rendered by `AppShell` above `<Outlet />`. Error = red; closed = mute with reconnect button.
- **Per-session errors** (`session_dead`): existing `transcriptOnly` flow. Banner inside Chat header area; ResumePrompt logic unchanged.
- **localStorage failures** (theme, projects): `try/catch` around access; fall back to in-memory state. Silent degrade. Theme defaults to `system`; projects to empty list.
- **Reduced motion:** `useReducedMotion()` from motion. When true: skip view-transition, sheet, modal animations; sheet handle indicator becomes static.
- **Focus management:** `BottomSheet` and `Modal` trap focus; Esc closes; restore focus to opener. Reuse focus-trap pattern from existing `Session.tsx` mobile-nav-drawer code (then delete that code). ProjectPicker focuses working-dir input on open.
- **Tap targets ≥44px:** NavRail mobile tabs `min-h-[56px]`; Send/Stop `min-h-[44px]`; list rows `min-h-[56px]`. Every drag/swipe has a tap-equivalent (Projects row delete: long-press OR swipe OR kebab menu).
- **Keyboard:** NavRail focusable in source order. Cmd/Ctrl+Enter sends in InputBox (existing). ↑ on empty input opens prompt history (existing). Escape closes modal/sheet/autocomplete.
- **Empty states** for Home active sessions, Home history, Sessions, Projects, Profiles, Accounts. Mockup-style centered mute card.
- **Loading states:** Initial connect — NavRail visible, content shows skeleton card-rows with `animate-pulse`. Transcript fallback — restyled "Loading transcript…" message.
- **Theme flicker:** inline `<script>` in `index.html` sets `data-theme` on `<html>` before React mounts.

## Risks

- **Tailwind v4 + Vite 5:** confirm `@tailwindcss/vite` plays nice with current Vite 5. May require Vite 5 → 6 bump. Validate in phase 0.
- **`motion` package** (not `framer-motion`): newer, smaller. Verify SSR-irrelevance for this client-only app. Validate in phase 1.
- **Snapshot churn:** if snapshot tests exist (none discovered), expect regeneration during port phases.
- **Tests asserting on CSS class names:** rewrite to assert behavior or roles, not utility class strings.
